# 온라인 초파일 연등 달기 PRD

## 1. 제품 개요

온라인으로 초파일 연등을 달고 소원을 남길 수 있는 참여형 웹페이지를 만든다. 사용자는 비어 있는 연등을 선택해 이름, 소원 내용, 연등 색상을 입력하고, 입력된 연등은 메인 화면에서 색이 들어온 상태로 표시된다. 다른 사용자가 남긴 연등을 선택하면 해당 소원을 확인할 수 있다.

초기 구현은 UI 프로토타입과 프론트엔드 상태 관리에 집중한다. 데이터베이스 연결 전까지는 `localStorage`를 사용해 소원 데이터를 저장하고, 이후 Supabase로 교체할 수 있도록 데이터 저장 로직을 분리한다.

## 2. 제품 목적

- 오프라인 초파일 연등 달기 경험을 온라인에서 가볍게 재현한다.
- 사용자가 자신의 소원을 하나의 연등에 담아 남길 수 있게 한다.
- 여러 사람의 소원이 모여 있는 SNS형 연등 피드 경험을 제공한다.
- 향후 Supabase 연결을 통해 실제 다중 사용자 데이터 저장과 조회가 가능하도록 설계한다.

## 3. 핵심 콘셉트

- 도트 그림체 기반의 연등 행렬
- 최대 120개의 연등 슬롯
- 소원이 없는 연등은 회색
- 소원이 있는 연등은 사용자가 선택한 색상으로 표시
- 연등 선택을 통한 입력 및 확인 경험
- 마우스 오버 또는 터치 시 바람에 흔들리는 듯한 인터랙션
- light 모드와 dark 모드 제공
- 사용자는 한 번만 소원을 입력할 수 있음

## 4. 주요 사용자

초파일, 부처님오신날, 사찰 행사, 온라인 기원 이벤트에 참여하고 싶은 일반 방문자. 별도 회원가입 없이 짧은 시간 안에 자신의 소원을 남기고, 다른 사람들의 소원을 둘러볼 수 있는 사용자를 기준으로 한다.

## 5. 사용자 시나리오

### 5.1 첫 방문자

1. 사용자가 웹페이지에 접속한다.
2. 메인 화면에서 120개 이하의 연등 행렬을 본다.
3. 회색 연등 하나를 선택한다.
4. 소원 입력 팝업이 열린다.
5. 이름, 소원 내용, 색상을 입력한다.
6. 입력 완료 버튼을 누른다.
7. 수정 불가 안내와 함께 재확인 팝업이 열린다.
8. 사용자가 최종 확인하면 소원이 저장된다.
9. 선택한 연등이 색이 들어온 상태로 바뀐다.

### 5.2 이미 소원을 입력한 사용자

1. 사용자가 다시 회색 연등을 선택한다.
2. 시스템은 이미 소원을 입력한 사용자임을 확인한다.
3. 추가 입력을 막고 안내 메시지를 보여준다.
4. 사용자는 기존 소원이 입력된 연등을 눌러 소원을 확인할 수 있다.

### 5.3 다른 소원 확인

1. 사용자가 색이 들어온 연등을 선택한다.
2. 소원 확인 팝업이 열린다.
3. 팝업 상단에는 연등 아래쪽 모습이 보인다.
4. 중앙 소원지 영역에 이름과 소원 내용이 표시된다.
5. 돌아가기 버튼을 누르면 메인 페이지로 돌아간다.

## 6. 페이지 및 화면 구성

### 6.1 메인 페이지

메인 페이지는 연등들이 행렬을 맞춰 늘어서 있는 모습을 화면 가득 보여준다.

필수 요소:

- 연등 그리드
- 최대 120개 연등 슬롯
- light/dark 모드 전환 버튼
- 비어 있는 연등과 입력된 연등의 상태 구분
- 연등 hover/tap 애니메이션
- 입력 팝업
- 확인 팝업

연등 상태:

| 상태 | 표현 | 클릭 동작 |
| --- | --- | --- |
| 비어 있음 | 회색 연등 | 소원 입력 팝업 열기 |
| 입력 완료 | 선택 색상 연등 | 소원 확인 팝업 열기 |
| hover/tap | 좌우 흔들림 | 상태별 클릭 동작 유지 |
| dark 모드 입력 완료 | 은은한 발광 | 소원 확인 팝업 열기 |

### 6.2 소원 입력 팝업

회색 연등을 선택했을 때 열리는 입력 화면이다.

상단 영역:

- 스님 캐릭터 이미지
- 안내문

안내문 문구:

> 연등에 소원을 입력하세요.  
> 소원은 수정할 수 없으니 염원을 담아 신중히 입력하세요.

입력 필드:

- 이름
- 소원 내용
- 색 선택: 빨강, 초록, 파랑, 노랑

버튼:

- 입력 완료
- 닫기 또는 취소

입력 완료 전 재확인 문구:

> 소원은 한 번 입력하면 수정할 수 없습니다.  
> 이대로 연등에 소원을 담을까요?

검증 조건:

- 이름은 필수 입력
- 소원 내용은 필수 입력
- 색상은 필수 선택
- 소원은 입력 후 수정 불가
- 이미 소원을 남긴 사용자는 추가 입력 불가

### 6.3 소원 확인 팝업

색이 들어온 연등을 선택했을 때 열리는 확인 화면이다.

필수 요소:

- 상단 연등 아래쪽 이미지
- 소원지 형태의 중앙 영역
- 이름 표시
- 소원 내용 표시
- 돌아가기 버튼

동작:

- 돌아가기 버튼을 누르면 팝업이 닫히고 메인 페이지가 보인다.
- 팝업 외부 클릭 또는 ESC 키로 닫을 수 있다.

## 7. 반응형 요구사항

### 7.1 데스크톱

- 넓은 화면에서 여러 열의 연등 그리드를 표시한다.
- 화면 상단 또는 우측에 light/dark 모드 전환 버튼을 둔다.
- 팝업은 화면 중앙에 배치한다.
- hover 애니메이션을 적극적으로 사용한다.

### 7.2 모바일

- 5열 내외의 세로형 연등 그리드를 기본으로 한다.
- 세로 스크롤을 통해 전체 연등을 확인할 수 있다.
- 팝업은 모바일 화면 너비에 맞춰 표시한다.
- hover 대신 tap 애니메이션을 사용한다.
- 버튼과 색 선택 영역은 터치하기 쉬운 크기로 제공한다.

## 8. 비주얼 방향

전체 비주얼은 도트 그림체를 기준으로 한다.

필요 이미지:

- 회색 연등
- 빨간 연등
- 초록 연등
- 파란 연등
- 노란 연등
- dark 모드용 점등 연등 또는 발광 효과
- 연등 아래쪽 이미지
- 스님 캐릭터
- 소원지 이미지 또는 소원지 배경
- light 모드 배경
- dark 모드 배경

이미지가 준비되기 전에는 CSS 기반의 임시 도트 스타일로 구현할 수 있다.

## 9. 인터랙션 요구사항

### 9.1 연등 흔들림

- 마우스가 연등 위를 지날 때 연등이 좌우로 짧게 흔들린다.
- 흔들림은 바람에 살짝 흔들리는 정도로 자연스럽게 표현한다.
- 모바일에서는 터치 시 같은 애니메이션이 1회 재생된다.

### 9.2 모드 전환

light 모드:

- 밝은 배경
- 연등 색상은 보이지만 발광은 약함
- 회색 연등은 비어 있는 상태로 명확히 구분

dark 모드:

- 어두운 배경
- 입력된 연등은 은은하게 빛남
- 회색 연등은 불이 꺼진 상태처럼 표현

## 10. 데이터 요구사항

초기 구현은 `localStorage`를 사용한다. 이후 Supabase로 전환한다.

### 10.1 Wish 데이터 구조

```json
{
  "id": "string",
  "slotIndex": 0,
  "name": "string",
  "content": "string",
  "color": "red | green | blue | yellow",
  "createdAt": "ISO datetime"
}
```

### 10.2 로컬 저장 키

- `wishBuddha:wishes`: 전체 소원 목록
- `wishBuddha:participantId`: 현재 브라우저의 참여자 식별값
- `wishBuddha:submittedWishId`: 현재 브라우저에서 이미 입력한 소원 ID
- `wishBuddha:theme`: light/dark 모드

### 10.3 localStorage 정책

- 브라우저 단위로 소원 1회 입력을 제한한다.
- 저장된 소원 목록은 페이지 로드 시 불러온다.
- 새 소원이 입력되면 기존 목록에 추가하고 다시 저장한다.
- 같은 브라우저에서 이미 소원을 입력한 경우 추가 입력을 막는다.

### 10.4 Supabase 전환 시 고려사항

예상 테이블명: `wishes`

컬럼:

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | uuid | 소원 고유 ID |
| slot_index | integer | 연등 슬롯 번호 |
| name | text | 입력자 이름 |
| content | text | 소원 내용 |
| color | text | 연등 색상 |
| participant_id | text | 익명 참여자 식별값 |
| created_at | timestamptz | 생성 시각 |

제약:

- `slot_index`는 unique
- `participant_id`는 unique로 두어 1회 입력 제한
- `color`는 red, green, blue, yellow 중 하나

## 11. MVP 범위

MVP에 포함:

- 메인 연등 그리드
- 최대 120개 슬롯
- 소원 입력 팝업
- 소원 확인 팝업
- light/dark 모드
- 연등 hover/tap 애니메이션
- localStorage 저장
- 브라우저 단위 1회 입력 제한
- 반응형 UI

MVP에 미포함:

- Supabase 실제 연결
- 회원가입/로그인
- 관리자 페이지
- 소원 수정 기능
- 소원 삭제 기능
- 신고 기능
- 공유 기능

## 12. 향후 확장

- Supabase 연결
- 실시간 연등 업데이트
- 관리자용 소원 관리 페이지
- 부적절한 소원 신고 및 숨김 처리
- 특정 연등 URL 공유
- 행사 기간 종료 후 아카이브 페이지
- 방문자 수 또는 총 소원 수 표시
- 배경 음악 및 사운드 효과

## 13. 성공 기준

- 사용자는 1분 안에 소원을 입력할 수 있다.
- 입력 후 선택한 연등이 즉시 색상 상태로 반영된다.
- 색칠된 연등을 누르면 저장된 소원을 확인할 수 있다.
- 같은 브라우저에서는 소원을 한 번만 입력할 수 있다.
- 데스크톱과 모바일에서 모두 주요 흐름이 문제없이 동작한다.
- 이후 Supabase 저장소로 전환하기 위해 UI와 저장 로직이 분리되어 있다.
